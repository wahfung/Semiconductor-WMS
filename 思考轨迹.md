# 半导体仓库管理系统 - 开发思考轨迹

## 1. 需求分析阶段

### 1.1 核心需求理解
- **业务场景**: 半导体行业仓库管理，涉及原材料（硅片、光刻胶等）、半制品（晶圆各阶段）、产成品（芯片）
- **核心功能**: 采购、入库、出库、盘点四大核心业务流程
- **亮点需求**: 3D货架可视化展示，实时查看库存分布

### 1.2 技术选型决策
- **后端**: Spring Boot 3.2 - 企业级Java框架，成熟稳定
- **前端**: React + Ant Design - 组件丰富，开发效率高
- **3D可视化**: Three.js + @react-three/fiber - React生态下最佳3D方案
- **数据库**: MySQL 8.0 - 成熟的关系型数据库，支持复杂查询

## 2. 架构设计阶段

### 2.1 后端架构
```
采用经典三层架构：
Controller层 -> 接收请求、参数校验、返回响应
Service层 -> 业务逻辑处理、事务管理
Repository层 -> 数据访问、JPA查询
```

### 2.2 数据模型设计
```
核心实体关系：
Warehouse(仓库) 1:N Shelf(货架) 1:N ShelfSlot(货位)
Material(物料) 1:N Inventory(库存)
PurchaseOrder -> InboundOrder -> Inventory -> OutboundOrder
```

### 2.3 前端架构
```
组件层次：
App -> MainLayout -> Pages
状态管理：Zustand (轻量级)
路由：React Router v6
API层：Axios + 拦截器
```

```javascript
// 核心组件结构
Canvas
  ├── Scene (场景)
  │   ├── Lights (光照)
  │   ├── Floor (地面)
  │   ├── Grid (网格)
```

## 4. 质量保证阶段

### 4.1 数据一致性
- 密码统一使用BCrypt加密，哈希值一致
- 中文使用UTF8MB4编码，确保无乱码
- 初始化数据符合半导体行业真实场景

### 4.2 UI一致性
- 统一使用Ant Design组件
- 自定义主题色和样式变量
- 所有表单、表格、按钮样式统一

### 4.3 错误处理
- 后端全局异常处理
- 前端Axios响应拦截器
- 用户友好的错误提示

## 5. 关键决策记录

### 5.1 为什么选择JPA而非MyBatis？
- JPA对于标准CRUD操作更简洁
- 自动建表功能便于快速开发
- 关联查询通过注解配置更直观

### 5.2 为什么使用Zustand而非Redux？
- 轻量级，API简洁
- 无需繁琐的action/reducer样板代码
- 对于中小型项目足够使用

### 5.3 3D渲染性能优化考虑
- 使用React.memo减少不必要重渲染
- 货位使用简单几何体（BoxGeometry）
- OrbitControls限制缩放范围
